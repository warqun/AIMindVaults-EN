# 세션 종료 시 상태 갱신 (Mandatory)

> 모든 볼트에 동일 적용. 모든 에이전트 공통.

## _STATUS.md 갱신 (필수)

세션 종료 시, 작업한 볼트의 `_STATUS.md`를 **직접 갱신**한다.

갱신 항목:
- **Now**: 이번 세션에서 완료하거나 진행 중인 작업
- **Next**: 다음에 이어서 할 작업
- **Blocked**: 막힌 사항 (없으면 "없음")
- **Decisions**: 이번 세션에서 내린 결정 (있을 경우, `(YYYY-MM-DD)` 형식으로 추가)

추가로, 루트 `_STATUS.md`에서 해당 볼트 섹션도 갱신:
- Now / Last Agent / Next / Blocked

**`_STATUS.md` 갱신 없이 세션을 종료하지 않는다.**

## AGENT_STATUS 갱신 (권장)

각 에이전트의 개별 상태 파일(`AGENT_STATUS.md`)은 에이전트 자체 판단으로 갱신한다.
- 복잡한 작업, 다음 세션에서 이어받을 맥락이 있을 때 권장
- 단순 작업이면 생략 가능

## 설계 근거

- `_STATUS.md`는 사용자가 전체 현황을 파악하는 유일한 문서
- 4개 에이전트가 모두 사용되는 것은 아니며, 실제 운용은 1에이전트-1볼트 교대 방식
- AGENT_STATUS 병합 방식은 병합 시점 누락·규칙 미적용 문제로 폐지
- `_STATUS.md` 직접 갱신이 가장 확실하고 단순한 방법
