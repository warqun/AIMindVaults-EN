# Script-Creation Pre-Approval (Mandatory)

> Applied uniformly to every vault. Common to all agents.

## Rules

- If a task requires creating a new script (`.ps1`, `.py`, `.bat`, `.sh`, etc.), **report to the user and obtain approval before creating it.**
- Do not create or run a script without approval.

## Items to Report

1. Purpose of the script (why it is needed)
2. Creation path
3. Scope of impact when run (which files / folders are affected)
4. Whether it is one-shot or kept long-term

## Forbidden

- Do not create automation scripts behind the user's back.
- Do not modify existing scripts without authorization.
- Do not after-the-fact "I created and ran it" reporting without prior approval.
